# 运营闭环体系

*制定时间: 2026-03-03*

## 一、闭环流程

```
每日目标 → 执行 → 验证 → 汇报 → 复盘 → 优化
```

---

## 二、每日循环

### 08:00 晨间战略汇报
- 任务清单
- 服务状态
- 今日目标

### 12:00 午间检查
- 进度核查
- 问题发现

### 18:00 晚间复盘
- 成果汇报
- 明日计划

### 21:00 自检反思
- 卡点总结
- 经验沉淀

---

## 三、任务闭环

```
立项 → 分配 → 执行 → 验证 → 交付 → 归档
```

| 阶段 | 动作 |
|------|------|
| 立项 | 明确目标+产出 |
| 分配 | 责任人+时间 |
| 执行 | 实时产出 |
| 验证 | API测试/文档 |
| 交付 | 汇报+推送Git |
| 归档 | 更新memory |
