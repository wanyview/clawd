## 💊 Capsule 2: 元数据Schema

### 🔴 红色层 - 人类理解
```
【定义】
每个知识胶囊包含6类元数据

【内容】
1. 内容元数据: 原始文本、字数、时间戳
2. 主题元数据: 标签、置信度、层级
3. 概念元数据: 摘要、关键点、问答
4. 实体元数据: 人物、组织、地点、概念
5. 关系元数据: 相关胶囊、先验知识
6. 重要性元数据: 分数、复用概率

【用途】
结构化存储，支撑检索和发现
```

### 🔵 蓝色层 - AI原生
```
think_pattern:
  mode: structure_extraction
  process:
    - 接收原始文本
    - LLM分析提取6类元数据
    - 分类: topic/concept/entity/relationship/importance
    - 输出JSON结构

tools_needed:
  - llm_analyze
  - embedding_create

action:
  function: extract_metadata(text) → metadata_schema

reasoning:
  logic: 信息提取类 → 调用LLM → 结构化输出
```

```

capsule_id: kc_002
```

## 💊 Capsule 3: 生成过程

### 🔴 红色层 - 人类理解
```
【定义】
知识胶囊三阶段生成流程

【Stage 1: 内容处理】
• 清洗和标准化文本
• 分割为逻辑单元(段落、章节)
• 提取基础元数据

【Stage 2: AI分析】
• LLM主题分类
• 生成概念摘要
• 提取命名实体
• 识别关系
• 评估重要性

【Stage 3: 胶囊组装】
• 组合原始内容与元数据
• 创建向量嵌入
• 建立关系链接
• 存入知识库
```

### 🔵 蓝色层 - AI原生
```
think_pattern:
  mode: multi_step_pipeline
  process:
    - Step1: 预处理 - 清洗、分块
    - Step2: 分析 - 调用LLM提取
    - Step3: 组装 - 构建胶囊
    - Step4: 索引 - 向量化存储
  parallel: false
  iterations: 1-3

tools_needed:
  - text_cleaner
  - llm_analyze
  - embedding_service
  - vector_database

workflow:
  pipe(clean→analyze→assemble→index)

error_handling:
  if step_failed: retry or skip
```

## 💊 Capsule 5: 检索引擎

### 🔴 红色层 - 人类理解
```
【混合检索系统】

权重分配:
• 语义嵌入: 50% - 理解语义相似度
• BM25关键词: 20% - 精确匹配
• 图关系检索: 20% - 发现关联知识
• 时序相关: 10% - 最新优先

【检索流程】
1. 接收自然语言query
2. 并发执行4种检索
3. 结果融合与重排序
4. 返回Top-K结果
```

### 🔵 蓝色层 - AI原生
```
think_pattern:
  mode: hybrid_retrieval
  process:
    - 接收query
    - 并发: semantic + keyword + graph + time
    - 融合结果
    - rerank
    - 返回top-k
  parallel: true
  latency_target: <100ms

tools_needed:
  - semantic_search (weight: 0.5)
  - keyword_search (weight: 0.2)
  - graph_db (weight: 0.2)
  - time_rank (weight: 0.1)
  - reranker

reasoning:
  strategy: "先广后精，混合融合"
```
