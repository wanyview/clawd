## 一、任务流转机制

```
立项 → 分配 → 执行 → 检查 → 复盘 → 归档
```

| 阶段 | 负责 | 产出 |
|------|------|------|
| 立项 | 曹操(CEO) | 任务定义、验收标准 |
| 分配 | 各牵头人 | 责任人、时间节点 |
| 执行 | 责任人 | 阶段性产出 |
| 检查 | 赵云(东向) | 进度报告 |
| 复盘 | 曹操 | 经验总结 |
| 归档 | - | 文档沉淀 |

## 三、检查机制

### 每日检查 (08:00, 18:00)
- 检查当日任务进度
- 更新状态标记
- 报告阻塞问题

### 每周一 (10:00) 周会
- 回顾上周任务完成情况
- 复盘经验教训
- 制定本周计划

### 每月1日 (14:00) 月度会
- 月度总结
- 目标对齐
- 资源调配

---

## 四、验收标准

每个任务必须有:
1. 明确的产出物
2. 可衡量的指标
3. 截止时间
4. 责任人

---

## 五、状态定义

| 状态 | 含义 |
|------|------|
| ⏳ 待开始 | 已立项，未开始 |
| 🔄 进行中 | 正在执行 |
| 🔄XX% | 有具体进度 |
| ✅ 完成 | 已交付 |
| ❌ 阻塞 | 需要帮助 |

---

## 六、闭环流程

1. **立项**: 任务定义 → 验收标准
2. **分配**: 责任人 → 截止时间
3. **执行**: 产出 → 进度更新
4. **检查**: 东向监督 → 质量把关
5. **复盘**: 经验 → 改进
6. **归档**: 文档 → 知识库
